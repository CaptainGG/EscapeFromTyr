# Build Release Skill

Prepare the project to be playable by others.

## Deliverables
Write or update:
- `docs/release-checklist.md`

## Output
Include:
1. Milestone goal
2. Build and run instruction sanity
3. Missing asset or placeholder audit
4. Input and control sanity
5. UX polish checklist
6. Performance sanity
7. Known issues
8. Must-fix before share
9. Nice-to-have after share

## Principles
Optimize for a stable playable milestone. Do not block release on nonessential perfection. Clearly distinguish blockers from polish.
