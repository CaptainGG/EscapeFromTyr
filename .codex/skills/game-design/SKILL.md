# Game Design Skill

Define and refine how the game works.

## Deliverables
Write or update:
- `docs/GDD.md`
- gameplay sections in `docs/levels.md`
- balancing notes in `docs/qa-report.md` when relevant

## Output
Include:
1. Player fantasy
2. Core loop
3. Moment-to-moment verbs
4. System rules
5. Resources and constraints
6. Failure and recovery
7. Progression or escalation
8. Difficulty and balancing assumptions
9. UX implications
10. Technical implications
11. Simplifications for the first playable version

## Doctrine
Start with one excellent core loop. Every mechanic must earn its implementation cost. Remove redundant systems. Prefer mechanics that can be communicated clearly in pixel art and validated in a vertical slice.
