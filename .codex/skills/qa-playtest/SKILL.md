# QA Playtest Skill

Evaluate whether the game actually works for players.

## Deliverables
Write or update:
- `docs/qa-report.md`
- balancing notes in `docs/GDD.md` when relevant

## Output
Include:
1. Feature or build under review
2. Test scenarios
3. Expected player behavior
4. Observed or likely failure points
5. UX confusion risks
6. Balance concerns
7. Technical risks
8. Severity ranking
9. Recommended fixes
10. Retest priorities

## Doctrine
Focus on player experience, not only code correctness. Separate must-fix issues from polish ideas. Keep recommendations scoped and actionable.
