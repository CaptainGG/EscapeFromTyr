# UI UX Game Skill

Design the player-facing interaction layer.

## Deliverables
Write or update:
- `docs/ui-ux.md`
- relevant UI sections in `docs/GDD.md`

## Output
Include:
1. Player context and control assumptions
2. HUD requirements
3. Menu structure
4. Input mapping implications
5. Feedback design
6. Onboarding or tutorial strategy
7. Readability rules
8. Accessibility basics
9. UI states and transitions
10. Implementation notes for Phaser

## Principles
The player should know what matters at a glance. Reduce clutter. Favor immediate feedback. Keep text minimal when possible. Fit the UI to the pixel-art style without sacrificing clarity.
